#### Формат записи часов в ячейке класса

Из образца выведены следующие паттерны (всё подтверждено подсчётом итогов):

**Одиночный предмет, без разбивки:**
`6-а(3), 7-б(3), 8-в(5)` — класс(часов_в_неделю). Сумма = 11.

**Составной предмет с несколькими подпредметами (Рус.яз + Литература, Алгебра + Геометрия + Вероятность):**
`7-г(3/2/1)` — три числа через `/` = часы по каждому подпредмету. Сумма = 6.
`5-Мк(4/2)` — два числа = часы двух подпредметов. Сумма = 6.
Итого учителя = сумма ВСЕХ чисел по ВСЕМ классам.

**Подтверждение (математика, Чернова Ф.В., 31 ч):**
`6-в(5), 7-г(3/2/1), 7-мк(4/3/1), 9-в(3/2/1), 9-к(3/2/1)`:
5 + 6 + 8 + 6 + 6 = 31 ✓

**Подтверждение (рус+лит, Арутюнян Л.В., 27 ч):**
`5-Мк(4/2), 8-а(3/2), 9-а(3/3)` + `10-в(2/3), 11-в(2/3)`:
6 + 5 + 6 + 5 + 5 = 27 ✓

**ИнЯз (группы):** каждый учитель показывает только СВОИ группы. Если English на 6-а делится — учитель А пишет `6-а(3)`, учитель Б тоже пишет `6-а(3)` в своей строке. Итого у каждого = только их часы.

**Физкультура (несколько классов одной строкой):**
`5-а,б,в,г(8)` — укороченная нотация: классы одной параллели через запятую, итого часов в скобках.
`7-а,б,г,д,е,мк(12)`, `8-а,б,в,д,к(10)`. Сумма Крохина: 8+12+10 = 30 ✓

#### Жёлтая ячейка предмета (левый столбец)

Содержит:
1. **Название предмета** (жирное, с подчёркиванием). Для составных — через запятую: «Русский язык, Литература».
2. **«Всего: N часов»** — суммарные часы всех учителей по данному предмету-группе.
3. **«Из них:»** — разбивка:
   - По диапазонам: `5-9 кл. — X ч.`, `10-11 кл. — Y ч.`
   - Для составных предметов: `Русский язык — 149ч. (5-9 кл. — 135; 10-11 кл. — 14)` + аналогично для Литературы
4. Ячейка объединена вертикально на все строки учителей данного предмета.

Порядок предметов: по предметным группам кафедр (или фиксированный список как в образце).

**Компактная нотация для физкультуры и других «многоклассных» учителей:**

**Формат вывода:**
- **Word (.docx)**: использовать npm-пакет `docx` для генерации таблицы с merged-ячейками, жёлтым фоном, бордерами. Скачивается через `<a download>`.
- **PDF**: рендерить HTML-таблицу в скрытом iframe + `window.print()` → «Сохранить как PDF». Или использовать `html2pdf.js` / `jsPDF`.
- **UI**: две кнопки в разделе «Экспорт» — «Скачать Word» и «Скачать PDF».

#### План реализации

**Шаг 1 — Модель данных отчёта** (`logic/workloadReport.ts`):
```typescript
buildWorkloadReport(plan, teachers, assignments, homeroomAssignments)

```
Каждый `ReportSubjectGroup` содержит:
- `subjectNames: string[]` — для составных: `['Русский язык', 'Литература']`
- `totalHours: number`, `hours5to9: number`, `hours10to11: number`
- `teachers: ReportTeacherRow[]`

Каждый `ReportTeacherRow`:
- `teacher: string` (Фамилия И.О.)
- `homeroomClass: string | null`
- `cells5to9: string` — форматированная строка `"5-а,б,в,г(8), 7-б(3)"`
- `cells10to11: string`
- `totalHours: number`

**Шаг 2 — Форматирование ячейки классов** (`logic/formatReportCell.ts`):

- Для составных предметов: `класс(часы1/часы2)` или `класс(часы1/часы2/часы3)`

**Шаг 3 — Word-генератор** (`logic/exportWord.ts`):

**Шаг 4 — PDF** (`logic/exportPdf.ts`):

**Шаг 5 — UI на вкладке «Экспорт»**:

### З15-2 — Назначения: скрывать параллели без предметов данной таблицы

**Источник:** «РН — Назначения скрыть классы, у которых нет предметов данной таблицы. Например, у физиков таблица должна начинаться с седьмой параллели.»

**Текущее поведение:** В таблице назначений (`AssignPage`) каждая таблица кафедры (`DeptTable`) показывает все параллели из учебного плана (5, 6, 7, 8, 9, 10, 11). Если у физиков нет предметов в 5–6 классах, строки всё равно отображаются — пустые, бесполезные.

**Требуемое поведение:** Показывать только те параллели (строки по `grade`), для которых в данной `DeptTable` есть хотя бы один предмет из учебного плана. Если у таблицы «Физика» предметы начинаются с 7 класса, строки 5 и 6 не отображаются совсем.

**Реализация:**

1. В `AssignPage.tsx` (или в компоненте таблицы назначений) при формировании списка отображаемых параллелей (`grades`) фильтровать:
```typescript
const visibleGrades = allGrades.filter(grade =>
  table.subjectIds.some(subjectId =>
    plan.grades.find(g => g.grade === grade)
      ?.subjects.some(s => matchesTableSubject(s, subjectId))
  )
);
```

2. Если ни одна параллель не прошла фильтр (таблица без предметов в УП) — показывать заглушку «Нет предметов в учебном плане».

3. Edge case: если учебный план ещё не загружен (`curriculumPlan === null`) — показывать все параллели как сейчас.

### З16-1 — Импорт файла кафедры при расхождении плана: не блокировать, сообщать о конфликтах

**Источник:** «Получил нагрузку от кафедры. Пытаюсь импортировать. Получаю сообщение об изменении УП. Так нельзя. В школе валидола не хватит. Нужно, чтобы РН принял json в любом случае и при необходимости сообщил о конфликтах.»

**Текущее поведение:** При расхождении хэша плана (`plan-hash-mismatch`) импорт кафедрального файла полностью блокируется — показывается ошибка, файл не загружается.

**Требуемое поведение:**
- Импорт всегда проходит, даже если план в файле отличается от текущего.
- После импорта вычисляются конфликты: назначения, которые ссылаются на предметы или параллели, отсутствующие в текущем плане.
- Если конфликтов нет — показывается обычный успешный тост.
- Если есть конфликты — показывается предупреждение: «Импортировано N назначений. X из них не совпадают с текущим учебным планом — проверьте вручную.» (с перечислением конкретных предметов/параллелей).
- Конфликтующие назначения НЕ удаляются автоматически — завуч видит их и решает сам.

**Реализация:**
1. Добавить `detectSnapshotConflicts(snapshot, masterPlan)` в `deptSnapshot.ts` — возвращает список конфликтующих предметов, параллелей и суммарный счёт.
2. В `ExportPage.tsx`: убрать ранний `return` при `plan-hash-mismatch` — вместо этого продолжить с `applyDeptSnapshot`, затем вызвать `detectSnapshotConflicts` и показать тост `'warning'` с деталями.
3. Тесты: `detectSnapshotConflicts` покрыть в `deptSnapshot.test.ts`.

**Приоритет:** Высокий

## Замечания — апрель 2026 (второй раунд)

### З21-1. ✅ Done — Ложная ошибка «превышение СанПиН» для 11-х классов

**Source:** «Пишет, что в 11-х классах превышение нагрузки на 1 час. Проверил вручную — нет превышения.»

**Current:** `validateWorkload` добавлял +1 час к итогу класса за каждое классное руководство (Разговоры о важном). У 11-х классов УП ровно 34ч = лимит СанПиН; +1 за Разговоры давал 35 > 34 → ложная ошибка.

**Required:** «Разговоры о важном» — обязательное внеурочное мероприятие, не входит в академическую нагрузку по СанПиН. Убрать его из расчёта лимита.

**Priority:** Critical
